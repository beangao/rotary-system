# 要点
この会議では、小野が高にロータリー管理アプリの開発進捗と機能について詳細な説明を行った。小野は管理画面とアプリ画面の両方を画面共有しながら、会員管理機能、イベント管理、プッシュ通知機能などの仕様を詳しく説明した。アプリのユーザーフローについては、初回ユーザーと既存ユーザーで分岐し、メール認証とパスワード設定から始まって、会員名簿検索やイベント出席管理などの機能を紹介した。


# 次のステップ

- 小野: お知らせ管理および設定ファイル管理の画面を管理画面に追加し、高にファイルを送付する
- 小野: スーパーユーザー（全体管理ユーザー）用の管理画面構成について検討・実装を進める
- 小野: アプリ・管理画面の仕様書き出しや画面設計を必要に応じて進める
- 小野: 高と連携し、AIカスタマーセンター（管理画面）の進捗・要件について情報共有・調整を継続する
- 高: AIカスタマーセンターの管理画面（企業追加、エフェイ管理、データ管理等）実装を進める
- 小野: ログイン・会員登録・プロフィール編集等のアプリフローおよび管理画面の不具合・改善点を随時修正・調整する
- 小野: 他クラブ検索や交流機能など今後の拡張性を匂わせるUI（開発中ボタン等）を実装・維持する

# 要約

## 管理画面デザイン機能説明会

小野は高に管理画面のデザインと機能について説明し、会員管理、ダッシュボード、プッシュ通知機能を含むシステムの構成を共有した。システムには50名の既存ユーザーの一括登録機能が含まれており、会員のステータス（アクティブ、招待中、休会、大会）を管理できるようになっている。小野は会員へのメール送信機能についても説明し、特に招待メールとアクティブ化のための通知を送信する仕組みを紹介した。

## 会員・イベント管理システム説明会

小野は会員管理システムとイベント管理機能について説明し、イベント名、日時、場所、回答期限などの情報を登録できる仕組みを紹介した。アプリ側では出席または欠席の確認が可能で、プレビュー機能も利用できることが示された。小野はまだお知らせ管理機能を追加する必要があることを言及し、アプリ側のフローの説明を続ける予定であることが示された。

## アプリ実装計画共有会議

小野は複数のグループに対応するアプリの実装について説明し、最初に尼崎西グループで実証実験を行う計画を共有した。高は全体管理用のスーパーユーザー機能の必要性を確認し、小野は他のクラブとの交流機能をMVPでは実装しないが、将来的には検索機能を追加する計画を述べた。小野はアプリの初回起動フローについて、初回ユーザーと既存ユーザーで流れを分岐させる仕組みを説明した。

## パスワード再設定とアプリ機能説明

小野は、ログインに失敗したユーザー向けのパスワード再設定フローについて説明し、メールアドレス入力、認証コード入力、パスワード設定、プロフィール設定の手順を詳しく紹介した。アプリのメイン画面では、通知、例会、出席管理、会員名簿機能が含まれており、プライバシー設定やプロフィールの公開範囲についても言及された。小野は将来的な機能として、他メンバーのプロフィール検索機能の実装を計画していることを説明した。

## アプリ開発状況報告会

小野はアプリの開発状況について説明し、メンバーの絞り込み機能やプロフィールマッチング機能の実装状況を報告した。アプリは350人程度のメンバーや50人程度の小規模クラブに対応できるよう設計されており、タップ操作で検索や絞り込みを行えるようにしている。小野はパスワード再設定フローの説明を不足していたことを認め、イベントマネージメント機能についても説明を補足した。
